## Visão Geral

Dashboard administrativo completo para operações de e-commerce, desenvolvido para centralizar métricas de negócio, gestão de pedidos, controle de estoque e análise de clientes em uma única interface moderna e responsiva.

O projeto surgiu da necessidade de lojistas terem visibilidade em tempo real sobre seus negócios sem depender de ferramentas pagas e complexas. A solução foi construída como SPA (Single Page Application) com foco em performance e usabilidade.

## Funcionalidades Principais

- **Overview de KPIs**: Receita total, número de pedidos, ticket médio e taxa de conversão com comparativos por período.
- **Gestão de Pedidos**: Listagem paginada com filtros por status, data e cliente, com ações em lote (aprovar, cancelar, reembolsar).
- **Controle de Estoque**: Tabela de produtos com alertas de baixo estoque, histórico de movimentações e relatório de giro.
- **Análise de Clientes**: Segmentação por RFM (Recência, Frequência, Monetário), mapa de calor geográfico e cohort de retenção.
- **Gráficos Interativos**: Linha temporal de vendas, pizza de categorias e barras de performance por canal, com zoom e tooltip customizado.
- **Autenticação e Perfis**: Login com JWT, controle de acesso por roles (admin, gerente, estoquista) e log de auditoria.
- **Tema Claro/Escuro**: Suporte nativo a dark mode com persistência via localStorage.
- **Exportação de Relatórios**: Download de tabelas em CSV e PDF direto no browser.

## Tecnologias Utilizadas

- **React 18** com hooks modernos (`useTransition`, `useDeferredValue`) para UI responsiva
- **TypeScript** com tipagem estrita em toda a aplicação
- **TailwindCSS** + **Shadcn/UI** para componentes acessíveis e consistentes
- **Recharts** para gráficos interativos e responsivos
- **React Query (TanStack Query)** para cache de dados e sincronização com servidor
- **Zustand** para gerenciamento de estado global (carrinho, filtros, preferências)
- **React Hook Form** + **Zod** para formulários com validação declarativa
- **Vite** como bundler, com build otimizado e HMR instantâneo
- **Vitest** + **Testing Library** para testes unitários e de integração

## Arquitetura

O projeto segue a arquitetura **Feature-Sliced Design (FSD)**, separando responsabilidades em camadas (`app`, `pages`, `widgets`, `features`, `entities`, `shared`). Isso permite escalar o dashboard adicionando novos módulos sem alterar o código existente.

```
src/
├── app/          # Configurações globais, providers, rotas
├── pages/        # Composição das páginas (dashboard, orders, products...)
├── widgets/      # Componentes complexos autônomos (Sidebar, TopBar, DataTable)
├── features/     # Ações do usuário (auth, export, filter)
├── entities/     # Modelos de domínio (Order, Product, Customer)
└── shared/       # Utilitários, UI base, hooks compartilhados
```

## Desafios e Aprendizados

O maior desafio foi garantir performance em tabelas com milhares de registros. A solução foi combinar **paginação no servidor** com **virtualização de linhas** usando `@tanstack/react-virtual`, reduzindo o DOM de 10.000 para apenas ~30 elementos visíveis, o que resultou em 60fps constante mesmo em máquinas de entrada.

Outro ponto importante foi o design do sistema de permissões: implementei um hook `usePermissions()` que consulta as roles do JWT decodificado e expõe guards reutilizáveis, eliminando lógica de autorização espalhada pelos componentes.

- Tempo de carregamento inicial abaixo de **1.2s** (LCP) em conexão 4G
